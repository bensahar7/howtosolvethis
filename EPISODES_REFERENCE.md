## Show Format & Structure

Each episode follows a consistent **"Zoom In → Zoom Out"** structure:

### Standard Episode Arc

```
1. פתיח (Hook / Cold Open)
   — תיאור ויזואלי/עובדתי של הבעיה הגלובלית
   — שאלה מרכזית שמכוונת את הפרק

2. זום-אין לבעיה (The Problem Deep-Dive)
   — רקע מדעי/אקדמי (often with a researcher guest)
   — נתונים וסטטיסטיקות
   — למה זה קשה — constraints ו-tradeoffs

3. הפתרון (The Solution Segment)
   — הצגת החברה והטכנולוגיה
   — ראיון עם מייסד/מנכ"ל/CTO
   — איך זה עובד טכנית
   — מודל עסקי — מי משלם ולמה

4. זום-אאוט לשוק (Market Map)
   — סקירת נוף הפתרונות בתחום
   — שאלות פתוחות ועתיד הסקטור

5. Entrepreneur Insight (טיפ יזמי)
   — שאלה אחת אישית לאורח: מה הייתם אומרים ליזמים צעירים?

6. סיכום וקריאה לפעולה
   — תקציר הפרק
   — לינק לניוזלטר / ספוטיפיי / אפל
```

### Episode Types

| Type | Description | Examples |
|------|-------------|---------|
| **Single Company** | ראיון מעמיק עם מייסד אחד + חוקר | ep3, ep4, ep5, ep8, ep10, ep12–ep16 |
| **Multi-Company** | שני אורחים מחברות שונות עם גישות משלימות | ep1 (ToBee + BeeHero), ep2, ep6, ep7, ep9 |

### Metadata Format (meta.md.txt)

Each episode folder (`Context/Episodes/ep{N}-{slug}/`) contains a `meta.md.txt` with:

```
# Episode N: Title

## Core Metadata
- Guest, Role, LinkedIn
- Sector, Keywords
- Company Name, Website, Logo filename
- Researcher (optional): name, website, Google Scholar

## Multi-Company Episode (if applicable)
- Company Name 1/2, Logo, Website, Guest, Title, LinkedIn, Focus

## The Problem
## The Solution
## Key Discussion Points
## Entrepreneur Insight (quote)
## Growth Hierarchy Fields
- SEO Keywords
```
